Features

  • Created to simplify and accelerate the development of storage of any type: stores any sort of data.
  • No schema predefining or coding except for business logic
  • No SQL. Well... no SPARQL. No Erlang. Just dots and brackets. No user MapReduce functions
  • Queried like this: amount[z.usd.gross].val, person.name.cn.maiden, and that's it
  • Java-based: runs everywhere, where Java virtual machine runs. It is used as a simple and small Java library
  • Relational database is its data storage, almost any time-proven RDBMS may be used
  • Transactions, indices, in the nearest future - authorization subsystem, triggers
  • Run as an embedded storage, in the nearest future - as a server and/or as a web-service
  • Widely known interfaces, easy in programming

    • An internal way of storing data is a graph, which not only allows holding information with graph nature, like social networks or geo data. You can emulate storing tabular data as in relational DBs, but with

  • tables of variable widths,
  • columns of variant types, including
  • complicate structures
  • arrays of unlimited size
  • all of the above could be indexed and searched

    • Example

    This JSP code shows how we can output user information, given the user variable contains the needed object:

        <!-- user id is provided in id request parameter -->
    <c:set var="user" value="${z.by[param.id]}"/>

    <h1>User Info</h1>
    Login: ${user[z.login].val}<br/>
    Name: ${user[z.name].val}<br/>
    Created:
    <fmt:formatDate type="both" timeStyle="LONG" dateStyle="LONG" 
               value="${user[z.createDate].val}"/><br/>

    This shows how we can iterate over all users:

        <h3>List of Users:</h3>
    <c:forEach var="user" items="${z.users.set}">
      ${user[z.login].val}: ${user[z.name].val}<br/>
    </c:forEach>

    We may use JSTL forEach tag attributes features to organize paging etc.

    Here is shown how the form editing the user information may be defined:

        <!-- user id may be provided in id request parameter -->
    <c:set var="user" value="${z.by[param.id]}" scope="page"/>
    <!-- this actually processes the form submitted 
            and returns created user -->
    <c:set var="user" value="${metyForm.processed.user}" scope="page"/>

    <form method="POST" action="">
      <input type="hidden" name="set|z.users|user"
                 value="${user}">

      Login: <input type="text" name="put|user[z.login].val" 
                 value="${user[z.login].val}"/><br/>
      Name: <input type="text" name="put|user[z.name].val" 
                 value="${user[z.name].val}"/><br/>
      Password: <input type="password" 
                 name="put|user[z.password].val"/><br/>
      <input type="submit" value="Sign Up">
    </form>

    Beyond this, no schema and object mapping definition is required.
